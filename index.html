How to connect to this project:

1) Download .zip, put it anywhere

the project .zip is called team5midiLights-master.zip.
unzip and you get the folder team5midiLights-master

2) Open project team5midiLights-master in NetBeans

3)under Team at the top, select "Team >> Git >> initialize repository"

just click "ok" to put this in the same folder as the project (it the default)

4) under Team at the top, select "Team >> Commit.."

put your name in the Commit Message: window, then hit the "commit" button

5) under Team at the top, select "Team >> Remote >> Pull..

(NOT pull from upstream) the first choice selected is the local git
we want the second one "Specify Git Repository Location".

set the "Repository URL:" to https://github.com/hancharik/team5midiLights.git

hit next, under Remote Branches select master - origin>master

after the pull you will be asked to rebase or merge. for this first time, just rebase

6) Make a change to the code

change one of the teamMemberButtons on StartPanel.java line 370ish to your name.

(in teamMemberButtonx = new JButton("vacant") change "vacant" to your name) under Team at the top, select "Team >> Commit.."

under the commit message put "changed a name button", and commit.

7)under Team at the top, select "Team >> Remote >> Push..

(NOT push to upstream) the first choice selected is the local git

we want the second one "Specify Git Repository Location".

set the "Repository URL:" to https://github.com/hancharik/team5midiLights.git, >>next>>

Select Local Branches: select master -> master [U] >>next>>
Update Local References: select master -> origin/master [U] >>finish>>
That's it, you are good to go!

Team Five Final Project

We will be connecting the BES particle simulator to an external controller, midi and motion if time allows

Establishing a connection

MIDI is an obvious choice, as there are enough variables to allow for some interesting changes to the particles in real time. Connecting to a camera controller such as Microsoft Kinect Book on open source programming for the Kinect or using the camera on an Android phone would allow the user to control the light show with their hands (We only need to be able to access control for the mouse listener and a click event). Shining the screen at maximum brightness in a dark room could make for some interesting effects, although the dark room might require use of the accelerometer.

Why MIDI?

controlling the particle variables in real time with midi will be the main goal. It will be a good environment for running tests (4 midi channels and 5-10 variables) MIDI isn't about music, it's about multi-variable controllers. If we can connect to these particles, we can connect to any variable in any program, or even an airframe or robotic arm.
